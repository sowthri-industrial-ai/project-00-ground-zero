GenAI Architect Portfolio · Charter-00 · Ground Zero

Platform before payload.

Ground Zero is the architectural substrate on which every future project rests. Its value is not in what it does — it is in what it enables every subsequent project to do, faster and with zero architectural drift. Twenty minutes from empty Azure subscription to deployed demo, or the Charter doesn't close.

Duration 21 days · 5 waves Budget $19.63 of $25 ceiling Targets Local · Git · HF · Azure Dammam 2026-04-24
§ 01 · Mission

Prove the platform works end-to-end before any use-case content exists.

Ground Zero succeeds if, at closure:

  • A hiring-panel reviewer lands on the PCP, sees live architecture, clicks through to a working demo, views the as-built ledger, and trusts that what they see matches reality;
  • The architect can scaffold a new project with ./scripts/new-project.sh <n> and have CI, tests, deploys, and PCP integration work on first run;
  • A fresh Azure subscription, GitHub account, and HF account reproduce the entire system in under 20 minutes.
Load-bearing idea Most portfolios bundle infra into use-case projects. Ground Zero isolates the substrate so the substrate can be rigorous and the use-cases can breathe. This is how real enterprise GenAI programs work — MLOps before product.

§ 03 · System Zones

Four concentric zones. OIDC boundary around all of them.

OIDC federation surrounds the entire system. No Azure credentials ever stored in GitHub. Tokens minted per workflow run, scoped per run. The PCP sits intentionally outside the runtime substrate — monitoring must be independent of monitored (ADR-0003).

§ 05 · Stage Gates

Four gates. Fail-closed. Every gate ledger-written.

G3
Post-deploy
Per target
Smoke · endpoint reachable
Marked UNHEALTHY

Post-validation
All targets deployed

Each gate runs automated and writes a ledger entry. Ledger entries appear on the PCP As-Built timeline — including failures, by design. A portfolio that shows only green is a portfolio that has hidden its history.

§ 06 · Test Pyramid

Seven tiers. Responsible AI is first-class.

Most portfolios implement Unit + maybe Integration. Ground Zero implements seven. AI Guardrail and Responsible AI are the architect-signal differentiators — they test that safety boundaries actually fire, not that the code "looks" safe.

Tier 1 · Unit
Pure component logic · pytest · mocked dependencies · G1
Tier 2 · Smoke
Post-deploy health per target · G3
Tier 3 · Functional
End-to-end scenarios · Playwright + httpx · G4
Tier 4 · AI Guardrail
NeMo · Content Safety · Presidio assertions · G4
Tier 5 · Responsible AI ⭐
Bias · toxicity · hallucination · groundedness thresholds · Giskard + DeepEval · G4
Tier 6 · Red-team
Adversarial · Garak + PyRIT · scheduled weekly
Tier 7 · Eval Regression
RAGAS · DeepEval · Promptfoo · nightly threshold gate
Confidence signal Three permanent branches (demo-fails-g1 · demo-fails-guardrail · demo-regresses-eval) exist specifically to break. Each one demonstrates a different gate rejecting a different failure mode. Showing known failure modes is more convincing than polishing them away.

§ 08 · Architectural Decisions

Thirteen decisions · load-bearing.

ADR-0001
Azure region = Sweden Central · Global Standard
Microsoft-recommended · broadest EU model availability · ~100ms from Dammam · UAE North routes inference out
ADR-0002
OIDC federation GitHub ↔ Entra ID
Zero long-lived secrets · scoped tokens per workflow · architect signal for modern cloud security
ADR-0003
PCP on GitHub Pages · not Azure SWA
Monitoring must be independent of monitored · PCP on Azure creates circular dependency during the exact outages the PCP exists to report
ADR-0004
Polyrepo · hub + one repo per project
Hiring managers browse one repo at a time · independent CI/CD · each project sells on its own README
ADR-0005
deploy.sh is orchestrator · PCP is consumer-of-deploys
Failure isolation · broken site ≠ broken deploy
ADR-0006
Bicep primary · Terraform secondary
Azure-native speed + multi-cloud fluency signal
ADR-0007
Seven test tiers · Responsible AI + Red-team first-class
Most portfolios omit them entirely · architect-signal differentiator
ADR-0008
GitHub is source of truth · P-SelfDescribing principle
Eliminates the two-system problem that causes doc sites to drift within weeks
ADR-0009
Python 3.12 + uv
Current stable · fastest modern package manager · uv adoption signals currency
ADR-0010
Astro 5 for PCP
Markdown-native · zero-JS default · content collections map to project descriptors
ADR-0011
Deliberate failure-demo branches kept permanently
Showing known failure modes signals you understand the boundaries · confidence signal over polish
ADR-0012
PCP URL = <user>.github.io · free tier
$0/yr · content carries signal · portable later via CNAME
ADR-0013
Azure endpoint ephemeral by design
$0.50/demo vs $200/2-weeks if always-on · doubles as DoD #10 re-runnability proof
